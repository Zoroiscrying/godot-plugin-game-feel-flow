# Game Feel Flow 示例场景设计文档

> **设计目标：** 创建一个综合性的示例场景系统，用于演示、测试和教学Game Feel Flow插件的所有功能。

---

## 1. 整体架构

### 1.1 文件结构

```
addons/game_feel_flow/examples/
├── main.tscn                    # 主场景（导航+设置+帮助）
├── main.gd
├── scenes/
│   ├── effects_demo.tscn        # 效果演示子场景
│   ├── effects_demo.gd
│   ├── game_scenes.tscn         # 游戏场景子场景
│   ├── game_scenes.gd
│   ├── param_adjuster.tscn      # 参数调节子场景
│   ├── param_adjuster.gd
│   ├── combo_effects.tscn       # 组合效果子场景
│   ├── combo_effects.gd
│   ├── perf_monitor.tscn        # 性能监控子场景
│   └── perf_monitor.gd
├── components/                  # 可复用组件
│   ├── effect_card.tscn         # 效果卡片组件
│   ├── effect_card.gd
│   ├── param_panel.tscn         # 参数面板组件
│   ├── param_panel.gd
│   ├── code_preview.tscn        # 代码预览弹窗
│   └── code_preview.gd
└── resources/                   # 资源文件
    ├── themes/
    │   └── dark_theme.tres      # 暗色主题
    └── presets/
        └── default_params.tres  # 默认参数预设
```

### 1.2 架构模式

采用**主场景+子场景模块化**架构：
- 主场景负责导航、设置和全局状态管理
- 子场景负责具体功能演示
- 组件库提供可复用的UI组件

## 2. 主场景设计

### 2.1 布局结构

```
┌─────────────────────────────────────────────────────────────┐
│  顶部工具栏：2D/3D切换 | 自动演示控制 | 帮助按钮            │
├──────────┬──────────────────────────────────────────────────┤
│          │                                                  │
│  导航    │              内容区域                             │
│  菜单    │         （加载子场景）                            │
│          │                                                  │
│  - 效果  │                                                  │
│  - 游戏  │                                                  │
│  - 参数  │                                                  │
│  - 组合  │                                                  │
│  - 性能  │                                                  │
│  - 设置  │                                                  │
│          │                                                  │
├──────────┴──────────────────────────────────────────────────┤
│  底部状态栏：当前场景 | 活跃效果数 | FPS                     │
└─────────────────────────────────────────────────────────────┘
```

### 2.2 功能说明

**顶部工具栏：**
- `2D/3D切换按钮`：切换2D和3D场景视图
- `自动演示控制`：播放/暂停、上一个/下一个、循环模式
- `帮助按钮`：显示帮助信息弹窗

**导航菜单：**
- 使用ItemList或Tree控件
- 点击菜单项加载对应子场景到内容区域
- 高亮当前选中的菜单项

**底部状态栏：**
- 显示当前加载的子场景名称
- 显示当前活跃的效果数量
- 显示实时FPS

### 2.3 场景切换逻辑

```gdscript
# main.gd 核心逻辑
var current_scene: Node = null
var scenes: Dictionary = {
    "effects_demo": preload("res://addons/game_feel_flow/examples/scenes/effects_demo.tscn"),
    "game_scenes": preload("res://addons/game_feel_flow/examples/scenes/game_scenes.tscn"),
    "param_adjuster": preload("res://addons/game_feel_flow/examples/scenes/param_adjuster.tscn"),
    "combo_effects": preload("res://addons/game_feel_flow/examples/scenes/combo_effects.tscn"),
    "perf_monitor": preload("res://addons/game_feel_flow/examples/scenes/perf_monitor.tscn"),
}

func load_scene(scene_name: String) -> void:
    if current_scene:
        current_scene.queue_free()
    current_scene = scenes[scene_name].instantiate()
    content_container.add_child(current_scene)
```

## 4. 核心组件设计

### 4.1 效果卡片组件（EffectCard）

**属性：**
- `effect_name: String` - 效果名称
- `effect_type: String` - 效果类型
- `complexity: String` - 复杂度（simple/medium/complex）
- `preview_target: Node` - 预览目标节点

**方法：**
- `set_effect(name, type, complexity)` - 设置效果信息
- `start_preview()` - 开始预览
- `stop_preview()` - 停止预览
- `get_code_example() -> String` - 获取代码示例

**信号：**
- `clicked(effect_name)` - 点击事件

### 4.2 参数面板组件（ParamPanel）

**属性：**
- `effect_name: String` - 当前效果名称
- `params: Dictionary` - 当前参数值

**方法：**
- `setup_for_effect(effect_name)` - 为指定效果设置参数控件
- `get_params() -> GFFParams` - 获取当前参数
- `reset_params()` - 重置参数为默认值

**信号：**
- `params_changed(params)` - 参数变化事件

### 4.3 代码预览组件（CodePreview）

**属性：**
- `code_text: String` - 代码文本
- `language: String` - 代码语言

**方法：**
- `show_code(code, language)` - 显示代码
- `copy_to_clipboard()` - 复制到剪贴板
